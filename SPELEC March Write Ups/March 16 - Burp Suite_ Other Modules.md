## **Decoder Overview**

## The Decoder module in Burp Suite provides powerful data manipulation capabilities for encoding, decoding, and hashing data. It supports multiple encoding formats such as Base64, URL Encoding, HTML Entities, ASCII Hex, Octal, Binary, and Gzip compression. Decoder also includes Smart Decode, which recursively attempts to decode data automatically until plaintext is obtained.

### **Key Takeaways**

* ## Used for encoding and decoding intercepted data.

* ## Supports hashing algorithms such as MD5, SHA-256, and MD4.

* ## Includes Smart Decode for automatic recursive decoding.

* ## Can convert between text and hexadecimal byte representations.

* ## Helpful for analyzing encoded payloads and transformed data during web application testing.

### **Important Features**

* ## **Encode/Decode Functions**

  * ## URL Encoding

  * ## HTML Encoding

  * ## Base64

  * ## ASCII Hex

  * ## Binary/Octal/Hex conversions

  * ## Gzip compression

* ## **Hashing**

  * ## Generate hashes such as MD5, SHA-1, SHA-256, and MD4.

  * ## Useful for integrity verification and password analysis.

* ## **Smart Decode**

  * ## Automatically detects and decodes encoded content.

* ## **Hex View**

  * ## Allows byte-level editing and analysis of data.

## **Hashing Overview**

## Hashing is a one-way cryptographic process that converts data into a unique signature called a hash. Even small changes in input produce completely different outputs. Hashes are commonly used for:

* ## Password storage

* ## File integrity verification

* ## Digital signatures

* ## Data validation

### **Key Hashing Concepts**

* ## Hashes are irreversible.

* ## Identical input always produces identical output.

* ## Different input produces a completely different hash.

* ## MD5 is deprecated for secure applications.

### **Common Hashing Algorithms**

* ## MD5

* ## SHA-1

* ## SHA-256

* ## MD4

* ## SHA-512

## **Sequencer Overview**

The Sequencer module in Burp Suite is used to analyze the randomness, or entropy, of tokens generated by web applications. These tokens are commonly used for:

* Session cookies  
* CSRF tokens  
* Password reset tokens  
* Authentication identifiers

A secure application should generate tokens using cryptographically secure randomization. Weak randomness can make tokens predictable, potentially allowing attackers to hijack sessions or bypass security controls.

### **Key Takeaways**

* Evaluates token entropy and randomness.  
* Helps identify weak or predictable token generation.  
* Commonly used to test session cookies and CSRF tokens.  
* Supports both live token capture and manual token loading.  
* Generates detailed randomness analysis reports.

---

## **Sequencer Capture Methods**

### **Live Capture**

* Sends repeated requests automatically.  
* Collects thousands of generated tokens from the server.  
* Best for real-time entropy analysis.

### **Manual Load**

* Imports previously collected token samples.  
* Reduces traffic sent to the target.  
* Useful for offline analysis.

---

## **Using Sequencer for Entropy Analysis**

### **Basic Process**

1. Capture a request in Proxy.  
2. Send the request to Sequencer.  
3. Select the token location:  
   * Cookie  
   * Form field  
   * Custom location  
4. Start live capture.  
5. Collect enough samples (around 10,000 recommended).  
6. Pause or stop capture.  
7. Analyze the collected tokens.

### **Additional Features**

* Auto Analyze option for periodic updates.  
* Ability to export captured tokens.  
* Statistical analysis of randomness quality.

## **Organizer Overview**

The Organizer module in Burp Suite helps store, organize, and annotate HTTP requests for later review during penetration testing.

### **Key Takeaways**

* Stores copies of interesting HTTP requests.  
* Helps organize testing workflow.  
* Requests can be sent from Proxy, Repeater, and other modules.  
* Supports note-taking and annotation.  
* Useful for reporting vulnerabilities later.

### **Important Features**

* Requests are stored in a sortable table.  
* Includes metadata such as:  
  * HTTP method  
  * URL  
  * Status code  
  * Response length  
  * Tool source  
  * Notes  
* Requests and responses are viewable directly inside Organizer.
